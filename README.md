### Подключение к потокам, остановка, изменение конфига и проблема с портом
![error](img/13.55.07.png)
#### Объяснение:
`docker attach` подключается к `PID 1` контейнера (nginx в foreground-режиме). Ctrl+C отправляет `SIGINT` главному процессу → nginx завершает работу → контейнер останавливается

![swapPort](img/13.56.04.png)
![swapPort](img/13.56.45.png)
#### Объяснение:
Port mapping (-p 127.0.0.1:8080:80) фиксирован на старте контейнера и перенаправляет на внутренний порт 80.
После изменения конфига nginx слушает 81 → трафик на внутренний 80 не обрабатывается → страница с хоста недоступна.
